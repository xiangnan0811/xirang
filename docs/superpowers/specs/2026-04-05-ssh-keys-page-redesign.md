# SSH Keys 页面重新设计

## 概述

SSH Keys 页面当前设计过于简陋——仅卡片视图、无搜索筛选、无批量操作、缺少实用功能——与 Nodes/Tasks 等页面风格不一致。本次重新设计采用 Nodes 页面的 UI 框架（表格/卡片双视图、FilterPanel、批量选择、分页），并加入 SSH 专属功能（密钥轮换向导、测试连接、批量导入/导出）。

## 设计方案

**方案 C：Nodes 框架 + SSH 专属特性**（已确认）

复用 Nodes 页面的成熟 UI 框架，在此基础上增加 SSH 密钥管理的专属功能。框架层面保持风格统一，功能层面体现专业性。

## 页面整体布局

### 统计卡片

页面顶部 4 张统计卡片，与 Nodes 页面风格一致：

| 卡片 | 数据源 | 样式 |
|------|--------|------|
| 总计 | `sshKeys.length` | 主色调（indigo） |
| 在用 | 有关联节点的密钥数 | 绿色 |
| 未用 | 无关联节点的密钥数 | 灰色 |
| 关联节点 | 所有密钥关联的节点总数 | 橙色 |

### 工具栏

左侧操作按钮 + 右侧视图切换：

**左侧**：
- **新建密钥**（主按钮）：打开现有 SSHKeyEditorDialog
- **批量导入**：打开批量导入对话框
- **导出公钥**：打开导出对话框
- **密钥轮换**：打开轮换向导
- **批量删除**（仅选中时显示）：删除选中的密钥，需确认

**右侧**：
- 表格/卡片视图切换按钮（与 Nodes 页面一致）

### 筛选栏

使用 FilterPanel 组件，包含：
- **关键词搜索**：匹配名称、用户名、指纹
- **密钥类型筛选**：全部 / RSA / ED25519 / ECDSA
- **使用状态筛选**：全部 / 在用 / 未用
- **排序**：名称 / 创建时间 / 最后使用时间

### 表格视图（桌面端默认）

表格列定义：

| 列 | 字段 | 说明 |
|----|------|------|
| 复选框 | — | 批量选择，表头有全选 |
| 名称 | `name` | 带密钥图标，字体加粗 |
| 用户名 | `username` | 次级文字颜色 |
| 类型 | `keyType` | Badge 展示（ED25519=indigo, RSA=amber, ECDSA=teal） |
| 指纹 | `fingerprint` | monospace 截断显示 |

**未使用密钥**：行文字降低不透明度（opacity: 0.6），"从未使用"斜体显示。

### 卡片视图（移动端自动 / 手动切换）

每张卡片包含：
- 顶部：密钥图标 + 名称 + 用户名 + 类型 Badge + ⋯ 菜单
- 中部：2 列元数据网格（指纹、最后使用时间）
- 底部：使用状态 Badge + 快捷操作按钮（复制公钥、测试连接）

## 密钥轮换向导

### 入口

- **工具栏按钮**：全局入口，从步骤 1 开始
- **行操作菜单**：预选当前密钥，直接进入步骤 2

### 4 步流程

**步骤 1 — 选择密钥**
- 单选列表，仅展示有关联节点的密钥（无关联的没有轮换意义）
- 每项显示：密钥名称、用户名、类型、关联节点数

**步骤 2 — 上传新密钥**
- 复用 SSHKeyEditorDialog 的表单字段：密钥名称（预填）、密钥类型、私钥输入
- 支持文本粘贴和文件上传（.pem / .key / .openssh）

**步骤 3 — 确认影响**
- 警告横幅：提示受影响节点数和注意事项
- 受影响节点列表：节点名称 + 在线状态
- 新旧指纹对比
- 确认按钮使用警告色（amber）

**步骤 4 — 执行结果**
- 成功横幅：密钥已更新
- 连通性验证结果：逐节点测试新密钥的 SSH 连接
  - ✓ 验证通过 / ⏭ 跳过（离线）/ ✗ 验证失败
- 离线节点提示用户上线后手动验证
- 新指纹展示

### 后端实现

- 调用现有 `updateSSHKey` API 更新私钥字段
- 后端自动重新计算指纹和公钥
- 节点的 `key_id` 引用不变（节点通过外键引用密钥，替换私钥后自动生效）
- 更新完成后自动对在线节点执行连通性验证（复用 test-connection API）
- 审计日志记录：轮换操作 + 旧指纹 → 新指纹 + 影响节点列表

## 测试连接对话框

- 节点选择器：多选，默认展示使用该密钥的关联节点
- 点击测试后，逐节点显示结果：
  - 成功：绿色 ✓ + 延迟（ms）
  - 失败：红色 ✗ + 错误信息（超时/拒绝/密钥不匹配）
- 仅测试 SSH 握手和密钥认证，不执行命令
- 支持"重新测试"按钮

### 后端需求

新增 API：`POST /api/v1/ssh-keys/:id/test-connection`
- 请求体：`{ "nodeIds": ["node-1", "node-2"] }`
- 响应：每个节点的连接结果（success/error + latency）
- 实现：使用现有 sshutil 包建立 SSH 连接，超时 10 秒

## 关联节点面板

- 侧滑抽屉（Sheet 组件），从右侧滑入
- 展示使用该密钥的所有节点列表：
  - 节点名称（可点击跳转到 Nodes 页面）
  - IP 地址 + 端口
  - 在线状态指示灯
- 无关联节点时显示空状态

## 批量导入

- 对话框，支持拖拽上传 JSON 文件
- JSON 格式：`[{ "name", "username", "keyType", "privateKey" }]`
- 上传后预览校验：
  - 格式验证（必填字段检查）
  - 重名检测（与现有密钥比对）
  - 逐项显示状态：✓ 有效 / ⚠ 名称已存在 / ✗ 格式错误
- 仅导入有效项，按钮文字显示有效数量
- 导入完成后 toast 提示 + 刷新列表

### 后端需求

新增 API：`POST /api/v1/ssh-keys/batch`
- 请求体：密钥数组
- 响应：逐项结果（created / skipped / error）
- 重名时跳过（不覆盖）

## 导出公钥

- 对话框，3 种导出格式：
  - **authorized_keys**：标准 SSH 格式，可直接部署到 `~/.ssh/authorized_keys`
  - **JSON**：结构化数据（name, username, keyType, publicKey, fingerprint）
  - **CSV**：表格格式，便于在电子表格中查看
- 3 种导出范围：
  - 所有密钥
  - 仅在用密钥
  - 当前选中（需先在表格中选择）
- 预览区显示前几行内容
- 点击下载按钮生成文件

### 后端需求

新增 API：`GET /api/v1/ssh-keys/export?format=authorized_keys|json|csv&scope=all|in_use&ids=...`
- 返回对应格式的文件内容
- Content-Disposition 头触发下载

## 公钥派生

当前 SSHKey 模型仅存储 `PrivateKey`，无 `PublicKey` 字段。复制公钥、导出公钥等功能依赖公钥数据。

**方案**：后端在 API 响应中增加 `public_key` 计算字段。

- 在 SSH key 的 GET/LIST 响应中，使用 Go 标准库（`crypto/x509`、`golang.org/x/crypto/ssh`）从私钥派生公钥
- 前端 `SSHKeyRecord` 类型新增 `publicKey?: string` 字段
- 派生在 API handler 层完成（不修改数据库模型），仅在需要时计算
- 导出 API 直接在后端完成派生，无需前端传入公钥

## 不在范围内

- 密钥自动过期/提醒机制
- 密钥权限分级（RBAC 控制哪些用户可操作哪些密钥）
- 远程服务器上的 authorized_keys 文件同步
- 密钥生成（在服务器端生成密钥对）
